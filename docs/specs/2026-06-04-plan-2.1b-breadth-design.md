# Design Spec — Plan 2.1b: Channel Creation, Thread Ordering, Message Search

**Status:** 🟢 Scoped (author's call); ready for planning
**Date:** 2026-06-04
**Parent:** [`2026-06-03-agent-chat-fusion-design.md`](./2026-06-03-agent-chat-fusion-design.md)
**Builds on:** Plan 2.1a (navigation), merged to `main`.

## 1. Goal
Add workspace breadth on top of 2.1a: **create channels**, **order threads chronologically**, and
**search messages**. DMs are the heaviest remaining 2.1 piece (they need a member/principal model) and
are split into **2.1c**.

## 2. Scope decisions (taken)
**In:** create a channel; add `threads.createdAt` + chronological (newest-first) thread ordering;
full-text-ish message search (`ILIKE`) scoped to the org, with jump-to-thread.
**Out (→ 2.1c+):** DMs, channel rename/archive, search ranking/highlighting/pagination, real auth.

## 3. Schema change
- Add `threads.createdAt timestamptz NOT NULL DEFAULT now()` (Drizzle migration). Existing rows
  backfill to migration time. `listThreads` orders by `createdAt DESC` (replaces the id-desc proxy).

## 4. API (additions to `services/app`)
Org-scoped via the `x-org-id` dev header.

| Method · Path | Returns | Notes |
|---|---|---|
| `POST /channels` `{name}` | `Channel` (201) | creates in the org's (first) workspace |
| `GET /search?q=<text>` | `SearchResult[]` | `ILIKE %q%` on `messages.body`, org-scoped, joined to thread title, newest-first, limit 50; empty `q` → `[]` |

`SearchResult = { messageId, threadId, threadTitle, body, kind, createdAt }`.

## 5. UI (additions to `services/web`)
- **`Sidebar`** gains a small "create channel" input (calls `onCreateChannel(name)`), and renders
  channels ordered.
- **`SearchBar`** (new): a text input that queries `/search` (debounce-free for 2.1a simplicity — query
  on submit/Enter), shows a results list (`threadTitle` + body snippet); clicking a result selects that
  thread. Lives in the main header.
- **`App`** wires `onCreateChannel` (POST then refresh channels) and search-result selection
  (`setActiveThreadId`). Created channels/threads refresh the sidebar.

## 6. Testing
- **Backend:** `createChannel` (finds workspace, inserts) + `searchMessages` (matches body, org-scoped,
  empty-q → []) module tests; routes via `app.inject` (`POST /channels` 201, `GET /search?q=` filters).
- **UI:** `SearchBar` (renders results from an injected search fn, click selects); Sidebar channel-create
  (calls `onCreateChannel`); App smoke still renders with the added wiring (fetches stubbed).

## 7. Definition of Done (2.1b)
App backend suite green (Postgres, incl. the new migration applied) + web suite green + build clean.
Live: you can create a channel, new threads list newest-first, and searching surfaces matching messages
that jump to their thread. DMs remain for 2.1c.
