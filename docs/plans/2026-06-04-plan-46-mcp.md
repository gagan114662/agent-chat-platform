> **For agentic workers:** REQUIRED SUB-SKILL: subagent-driven-development or executing-plans. Checkbox steps. TDD.

**Design (author's call):** conductor 0.1.0 + reload MCP + the curated catalog (#97). Let an agent run with **MCP servers** it's authorized for: the sandbox holds a small **built-in MCP catalog** (name → command/args + security tier), validates requested servers against a **default-deny allowlist** (`ACP_ALLOWED_MCP`, like the adapter authz #38; `money`/`irreversible`-tier servers always denied unless explicitly allowed), writes a claude-readable `.mcp.json` into the run workdir (provisioned post-quarantine, removed pre-commit like built-in skills #48), and the claude CLI loads it via `--mcp-config`. The server list threads from `agents.config.mcpServers` (like model/provider #58).

- [ ] **Step 1 — `mcp.go`:**
  - `mcpCatalogEntry = { command string; args []string; tier string }` (tier: `safe` | `sensitive` | `money`). A built-in `mcpCatalog` map of a few safe entries, e.g. `"filesystem"`, `"git"`, `"fetch"`, `"sequential-thinking"`, `"memory"` (tier `safe`), `"github"` (tier `sensitive`). (Commands are best-effort `npx -y @modelcontextprotocol/server-<x>`-style; refined later.)

  - `provisionMcpConfig(repoDir string, names []string) (func(), error)`: filter `names` to authorized catalog entries; if none → no-op cleanup; else write `repoDir/.mcp.json` = `{"mcpServers": { name: {command, args} }}` for each; return cleanup that removes the file. (Call AFTER quarantine, like skills.)

Add `McpServers []string` to `adapter.PrepareContext`; `http.go` passes `req.McpServers`.
- [ ] **Step 3 — adapter:** both `ClaudeCodeAdapter` + `CodexAdapter` capture `mcpServers` in `Prepare`; in the shared run path (`runAgentShared`), after `provisionBuiltinSkills`, call `provisionMcpConfig(repoDir, mcpServers)` (defer cleanup); when a config was written, append `--mcp-config <repoDir>/.mcp.json` to the CLI argv (claude).
