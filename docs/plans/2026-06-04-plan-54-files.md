## Task 0: files model + storage backend + signed URLs

**Files:** `services/app/src/db/schema.ts` + next migration (`0022_files.sql`), Create `src/files/storage.ts`, `src/files/files.ts`, `files.test.ts`
- [ ] **Step 1 — schema/migration:** `files` table: `id` (pk), `orgId`, `name`, `contentType`, `artifactKind` (text: `code|document|markdown|image|other`), `size` (integer, default 0), `storageKey` (text), `uploaded` (boolean default false), `uploadedByKind`, `uploadedById`, `createdAt`. (Attachments: store a `fileIds` array in message metadata, OR a `message_attachments` join — pick the simpler; the plan assumes message metadata + a list endpoint.) `pnpm db:migrate`.
- [ ] **Step 2 — `storage.ts`:** `StorageBackend` interface `{ put(key, data: Buffer): Promise<void>; get(key): Promise<Buffer>; }`. `LocalStorage(dir = process.env.ACP_STORAGE_DIR ?? <tmp>)` — writes/reads files under `dir` (mkdir -p, key sanitized — no traversal). Export `defaultStorage()` (local for now; doc the S3/R2 swap).
- [ ] **Step 3 — `files.ts`:**
  - `createFile(db, { orgId, name, contentType, size, byKind, byId })` → infer `artifactKind` from contentType/extension; `storageKey = \`${orgId}/${id}\``; insert (uploaded=false). Return the file.
  - `signFileUrl(fileId, op: "put"|"get", ttlSec = 600, now = Date.now())` → `\`${op}:${fileId}:${exp}\`` HMAC-signed with `ACP_FILE_SIGN_SECRET` (or a stable app secret) → a token; `verifyFileSig(fileId, op, token, now)` → boolean (timing-safe, expiry-checked).
  - `markUploaded(db, orgId, fileId, size)`; `getFile(db, orgId, fileId)`.
- [ ] **Step 4 — test:** `createFile` infers `artifactKind` (`.md`→markdown, `image/png`→image, `.ts`→code); `signFileUrl`+`verifyFileSig` round-trip true, tampered/expired → false; `LocalStorage` put/get round-trips a buffer; org-scoped. `DATABASE_URL=… pnpm test -- files` + tsc. Commit `feat(app): files model + storage backend + signed URLs (#80)`.

## Task 1: routes (create → signed upload → download → attach)

**Files:** Create `services/app/src/http/file-storage-routes.ts`, `file-storage-routes.test.ts`; Modify `src/server.ts`, `src/http/auth-routes.ts` (signed file routes can be bearer-auth OR the signed token — keep them under auth but allow the sig)
- [ ] **Step 1 — routes:** `registerFileStorageRoutes(app, d: { db, storage? })`:
  - `POST /files { name, contentType, size }` → `actor(req)`; size cap (e.g. 25 MiB → 400 over); `createFile`; return `{ file, uploadUrl: "/files/<id>/content?sig=<signPut>" }`.
  - `PUT /files/:id/content?sig=` → verify `sig` (put), read the raw body (cap), `storage.put(file.storageKey, body)`, `markUploaded`; 401 on bad sig, 413 over cap. (Raw body parser like the ingest route.)
  - `GET /files/:id` → metadata (org-scoped, 404).
  - `GET /files/:id/download?sig=` → verify `sig` (get), `storage.get(key)`, stream with the file's contentType; 401 on bad sig. (Returns `{ downloadUrl }` from `GET /files/:id` so clients fetch a signed link.)
  - Register in `server.ts`. (These routes: `POST /files` + `GET /files/:id` go through normal auth; the `?sig=` content/download routes verify the signature instead — add `/files/*` content/download to the preHandler bypass like `/ingest`, but enforce the sig.)
- [ ] **Step 2 — test** (`app.inject`, inject a fake/in-memory storage): `POST /files` → 201 with a file + uploadUrl; `PUT …/content?sig=<valid>` → 200 + stored + uploaded=true; bad sig → 401; oversize → 413/400; `GET /files/:id` → metadata + a signed downloadUrl; `GET …/download?sig=<valid>` → the bytes; cross-org `GET /files/:id` → 404. `DATABASE_URL=… pnpm test` + tsc. Commit `feat(app): file upload/download via signed URLs + attachments (#80)`.
